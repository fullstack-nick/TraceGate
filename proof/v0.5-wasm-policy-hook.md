# v0.5 WASM Policy Hook Proof

Date: 2026-07-06

## Scope

v0.5 implements a real pre-upstream `before_request` policy hook using Wasmtime Component Model.

Covered behavior:

- Configured `[[plugins]]` policy schema with explicit route bindings, raw-header allowlist, resource limits, fuel, timeout, body-preview limit, and plugin config values.
- WIT contract at `crates/tracegate-wasm/wit/policy.wit`.
- Startup plugin loading and contract/import validation.
- Pre-upstream allow/deny/header mutation/body-preview execution.
- Fail-closed plugin runtime timeout/error handling.
- Sanitized plugin decision persistence and CLI details output.
- Plugin Prometheus metrics.
- Bundled source-built example plugins copied into the Docker image.
- Local Docker and live GCP proof.

## Notes

- Raw sensitive headers remain omitted from storage. Plugins only see raw sensitive headers explicitly configured in a plugin `raw_headers` allowlist.
- The Rust `wasm32-wasip2` example components import minimal WASI CLI/IO shims even with `no_std`; TraceGate links those with an empty `WasiCtx` and no filesystem preopens, environment, or network access. Disallowed filesystem/network/HTTP/socket/random/clock imports are rejected by plugin import validation.
- The public VM endpoint is intentionally not recorded in this proof file.
