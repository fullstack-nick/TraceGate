## Scope Implemented

- Admin handling was refactored into a private Axum admin module while preserving:
  - `GET /health/live`
  - `GET /health/ready`
  - `GET /metrics`
  - `POST /admin/reload`
- The embedded console is served from the internal admin listener at `GET /console/`.
- Console assets are loadable without a bearer token and contain no embedded runtime data.
- Bearer-protected JSON endpoints were added under `/admin/api/*`:
  - `GET /admin/api/overview`
  - `GET /admin/api/requests?failed=&slow=&route=&limit=`
  - `GET /admin/api/requests/{request_id}`
  - `GET /admin/api/routes`
  - `GET /admin/api/plugins`
  - `GET /admin/api/telemetry`
- The console UI uses embedded HTML, CSS, and vanilla JS. It stores a manually entered admin token in `sessionStorage` only.
- The console displays recent requests, request details, replay runs, plugin decisions, route health, plugin summaries, and telemetry status.
- Replay remains CLI/script-only. The console displays persisted replay runs but does not initiate replay.
- Route-health snapshots are exposed from the existing gateway runtime state.
- Plugin API output includes plugin ids, route ids, limits, body-preview bytes, raw-header names, and config keys only. It does not expose config values.
- Grafana provisioning was added for the `TraceGate Overview` dashboard and wired into local demo, local production, and GCP Compose.
- Grafana exposure:
  - local demo: `localhost:3000`
  - local production: `localhost:13000`
  - GCP: Compose-network only, no public host port
- Full-demo scripts were added:
  - `scripts\full-demo.ps1`
  - `deployments\gcp\scripts\full-demo.ps1`
