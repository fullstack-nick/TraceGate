# TraceGate v1.0 Release Quality Proof

Date: 2026-07-07

Verdict: complete.

Runtime deployed to GCP: `tracegate:295ecd55dfbb`

Repository note: the deployed runtime commit is `295ecd55dfbb`. Later proof/tooling commits do not change the deployed Rust binary behavior unless explicitly redeployed.

## Scope Implemented

- `PLAN.md` v1 release-quality additions were committed first.
- Release-quality GCP mode was added while keeping `e2-micro` as the default steady-state app VM.
- Large VM guards were added:
  - app VM `n2-standard-16` requires release-quality mode
  - load generator `n2-standard-8` requires release-quality mode
  - cleanup returns the app VM to `e2-micro`
- Temporary load generator support was added with delete-after-v1 labeling, auto-delete boot disk, and no public app/admin ports.
- `deployments/gcp/scripts/v1-infra.ps1` was added for `BaselineUp`, `LoadGenUp`, `Cleanup`, and `Inventory`.
- `deployments/gcp/scripts/v1-live-verify.ps1` was added for full feature verification, backup, rollback, restart, reboot, and internal readback.
- `deployments/gcp/scripts/v1-stress.ps1` and `tests/load/k6/v1-stress.js` were added for Dockerized k6 stress from the load generator.
- CI, benchmark, benchmark docs, security docs, plugin authoring docs, and architecture docs were added during v1 hardening.
